# 🔐 Auth System v4.0

Система лицензирования и защиты программного обеспечения на базе **FastAPI + SQLite + C++ клиент**.

Позволяет распространять программу таким образом, что пользователь **никогда не получает исходный файл** — он скачивается с сервера зашифрованным и выполняется только в памяти, после успешной аутентификации и проверки железа.

---

## 📐 Архитектура

```
┌──────────────────────────────────────────────────────────┐
│                      auth_client.cpp                     │
│                                                          │
│  1. ECDH X25519 handshake → shared secret               │
│  2. AES-256-GCM шифрование всех запросов                │
│  3. Логин / регистрация (зашифровано)                    │
│  4. Пинг каждые 5 минут (сервер может отозвать доступ)  │
│  5. Запрос payload → расшифровка в памяти → запуск      │
└────────────────────┬─────────────────────────────────────┘
                     │  HTTPS / HTTP
┌────────────────────▼─────────────────────────────────────┐
│                    server/main.py (FastAPI)               │
│                                                          │
│  /handshake        — ECDH обмен ключами                  │
│  /secure/register  — регистрация (зашифровано)           │
│  /secure/login     — вход (зашифровано)                  │
│  /secure/ping      — проверка сессии                     │
│  /secure/launch    — выдача зашифрованного payload       │
│  /admin/*          — управление пользователями           │
└──────────┬────────────────────────┬──────────────────────┘
           │                        │
    ┌──────▼──────┐        ┌────────▼──────┐
    │  auth.db    │        │  Telegram Bot │
    │  (SQLite)   │        │  (уведомления)│
    └─────────────┘        └───────────────┘
```

## ✨ Ключевые особенности

### 🔒 Безопасность
- **ECDH X25519** — каждая сессия имеет уникальный ephemeral ключ. Перехват трафика не даёт ничего.
- **HKDF-SHA256** — деривация AES ключа из shared secret. Стандарт Signal Protocol.
- **AES-256-GCM** — authenticated encryption. Защищает от подмены данных.
- **PBKDF2-SHA256** с 260 000 итерациями — надёжное хранение паролей.
- **JWT токены** (HS256) с настраиваемым TTL.
- **HWID привязка** — программа привязывается к железу при первом входе. Передача логина другому человеку невозможна без сброса HWID администратором.

### 🛡️ Защита payload
- Файл приложения **никогда не сохраняется на диск** клиента.
- Payload скачивается зашифрованным, расшифровывается в RAM и сразу выполняется.
- Поддерживается запуск: **Python**, **Lua**, **Shell/Bash**, **Batch**, **EXE**.
- Сам payload дополнительно проверяет HWID через env-переменные (двойная защита).

### 👁️ Мониторинг
- **Telegram уведомления** в реальном времени: регистрация, вход, бан, ошибки, покупки.
- **Аудит-лог** всех событий в SQLite.
- **Ping-система** — сервер может мгновенно отозвать доступ (клиент это заметит на следующем пинге, максимум через 5 минут).

### 🛒 FunPay интеграция (опционально)
- Бот автоматически выдаёт инвайт-коды при покупке на FunPay.
- Уведомляет в Telegram о каждой покупке.

## 🚀 Установка сервера

### Требования
- Python 3.10+
- pip

### 1. Клонируй репозиторий и перейди в папку сервера

```bash
cd server/
```

### 2. Установи зависимости

```bash
pip install fastapi uvicorn[standard] pyjwt cryptography python-dotenv slowapi requests
```

Или через `requirements.txt`:

```bash
pip install -r requirements.txt
```

### 3. Создай `.env` файл

Скопируй `.env.example` и заполни:

```bash
cp .env.example .env
```

```ini
# Секретные ключи (можно оставить пустыми — сгенерируются автоматически в keys.json)
SECRET_KEY=         # JWT подпись. Если пусто — берётся из keys.json
ADMIN_KEY=          # Ключ для /admin/* эндпоинтов. ОБЯЗАТЕЛЬНО смени!

# Сервер
HOST=0.0.0.0
PORT=8000
JWT_EXPIRE_HOURS=24

# Telegram уведомления (опционально)
TG_TOKEN=           # Токен бота от @BotFather
TG_CHAT_ID=         # ID чата куда слать уведомления

# Payload (какой файл отдавать клиентам)
PAYLOAD_FILE=main/app.py   # Путь к файлу приложения

# FunPay (опционально)
FUNPAY_ENABLED=0
FUNPAY_GOLDEN_KEY=
```

### 4. Положи своё приложение в папку `payload/`

```bash
cp my_app.py server/payload/app.py
```

### 5. Запусти сервер

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

При первом запуске автоматически:
- Создастся `data/auth.db` (SQLite)
- Сгенерируются ключи в `data/keys.json`
- В консоль выведется `ADMIN_KEY` — **сохрани его**

### 6. Создай первый инвайт-код

```bash
curl -X POST "http://localhost:8000/admin/invite" \
     -H "x-admin-key: ВАШ_ADMIN_KEY"
```

## 🎯 Как это работает — шаг за шагом

```
Клиент                                    Сервер
  │                                          │
  │  GET /handshake                          │
  │─────────────────────────────────────────►│  Генерирует ECDH пару
  │◄─────────────────────────────────────────│  { session_id, server_pub_key }
  │                                          │
  │  Клиент генерирует свою ECDH пару        │
  │  Вычисляет shared_secret                 │
  │  AES_key = HKDF(shared_secret)           │
  │                                          │
  │  POST /handshake  { client_pub_key }     │
  │─────────────────────────────────────────►│  Вычисляет тот же AES_key
  │◄─────────────────────────────────────────│  { ok }
  │                                          │
  │  Все дальнейшие запросы зашифрованы AES  │
  │                                          │
  │  POST /secure/login [зашифровано]        │
  │─────────────────────────────────────────►│  Проверяет пароль, HWID
  │◄─────────────────────────────────────────│  { JWT_token } [зашифровано]
  │                                          │
  │  POST /secure/launch [зашифровано]       │
  │─────────────────────────────────────────►│  Проверяет JWT + HWID + бан
  │◄─────────────────────────────────────────│  { encrypted_payload }
  │                                          │
  │  Расшифровывает payload в RAM            │
  │  Запускает (python/lua/sh/exe)           │
  │                                          │
  │  POST /secure/ping  каждые 5 мин        │
  │─────────────────────────────────────────►│  Проверяет: не забанен? HWID ок?
  │◄─────────────────────────────────────────│  { ok } или 401/403 → клиент завершается
```

## 🛠️ Admin API

Все admin эндпоинты требуют заголовок `x-admin-key: ВАШ_ADMIN_KEY`.

### Пользователи

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/admin/users` | Список всех пользователей |
| DELETE | `/admin/users/{username}` | Удалить пользователя |
| POST | `/admin/users/{username}/ban` | Заблокировать |
| POST | `/admin/users/{username}/unban` | Разблокировать |
| DELETE | `/admin/users/{username}/hwid` | Сбросить привязку HWID |

### Инвайты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/admin/invite?count=5` | Создать инвайт-коды |
| GET | `/admin/invites` | Список всех инвайтов |

### Сессии и аудит

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/admin/sessions` | Активные сессии |
| GET | `/admin/audit?limit=200` | Аудит-лог |
| GET | `/admin/stats` | Статистика |

### Payload

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/admin/payload/upload` | Загрузить новый файл |
| GET | `/admin/payload/list` | Список файлов |
| DELETE | `/admin/payload/{filename}` | Удалить файл |

### Ключи

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/admin/keys/info` | Информация о ключах |

| POST | `/admin/keys/rotate?rotate_secret=true` | Ротировать SECRET_KEY ⚠️ сбрасывает все токены |

## 📦 Написание своего payload

### Python (рекомендуется)

Шаблон `payload/app.py`:

```python
import os, sys, time

AUTH_TOKEN  = os.environ.get("AUTH_TOKEN",  "")
AUTH_USER   = os.environ.get("AUTH_USER",   "")
AUTH_SERVER = os.environ.get("AUTH_SERVER", "")
AUTH_HWID   = os.environ.get("AUTH_HWID",   "")

# Обязательная проверка — без авторизатора не запустится
if not AUTH_TOKEN or not AUTH_HWID:
    sys.exit(1)

def main():
    print(f"Привет, {AUTH_USER}!")
    # === ПИШИ СВОЮ ЛОГИКУ ЗДЕСЬ ===

if __name__ == "__main__":
    main()
```

### Доступные env-переменные в payload

| Переменная | Описание |
|------------|----------|
| `AUTH_TOKEN` | JWT токен текущей сессии |
| `AUTH_USER` | Имя пользователя |
| `AUTH_SERVER` | Адрес сервера |
| `AUTH_HWID` | HWID текущей машины |

### Shell

```bash
#!/bin/bash
[ -z "$AUTH_HWID" ] && exit 1
echo "Привет, $AUTH_USER!"
# твоя логика
```

### Lua

```lua
local AUTH_HWID = os.getenv("AUTH_HWID") or ""
if AUTH_HWID == "" then os.exit(1) end
print("Привет, " .. (os.getenv("AUTH_USER") or "") .. "!")
-- твоя логика
```

## 🔄 Деплой в продакшн

### Запуск через systemd (Linux)

Создай `/etc/systemd/system/auth-server.service`:

```ini
[Unit]
Description=Auth Server
After=network.target

[Service]
User=www-data
WorkingDirectory=/opt/auth-server/server
ExecStart=uvicorn main:app --host 0.0.0.0 --port 8000 --workers 1
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

> ⚠️ Используй `--workers 1`. При нескольких воркерах ECDH сессии (хранятся in-memory) не будут работать между процессами.

```bash
systemctl enable auth-server
systemctl start auth-server
```

### HTTPS (рекомендуется для продакшна)

Через Nginx как reverse proxy:

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header X-Forwarded-For $remote_addr;
    }
}
```

Или напрямую через uvicorn с SSL:

```bash
uvicorn main:app --ssl-keyfile certs/server.key --ssl-certfile certs/server.crt
```

### 🔗 Активные сессии

Показывает всех пользователей, у которых есть активная сессия в БД.

Для каждой сессии отображается:
- Имя пользователя
- IP-адрес подключения
- HWID (первые 10 символов)
- Время создания сессии
- Время последнего пинга

> Сессия считается "активной" если токен ещё не истёк. Пользователь физически может быть уже оффлайн — пинг обновляется раз в 5 минут.

### Технические детали

- Консоль работает как **статический HTML файл** — не требует отдельного сервера
- Все данные загружаются напрямую с FastAPI через `fetch()`
- **Пинг сервера** каждые 15 секунд — индикатор онлайн/оффлайн в шапке
- **Автообновление** активной страницы каждые 30 секунд (можно отключить)
- Статистика карточек сравнивается с предыдущим визитом через `localStorage`
- Консоль не хранит Admin Key между сессиями — нужно вводить при каждом открытии

## ⚠️ Известные ограничения

| Проблема | Описание | Решение |
|----------|----------|---------|
| **Один воркер** | ECDH сессии в памяти, не шарятся между процессами | Запускать с `--workers 1` |
| **Бан не мгновенный** | JWT токен живёт до истечения (до 24ч) после бана | Уменьши `JWT_EXPIRE_HOURS` или добавь blacklist |
| **CORS открыт** | `allow_origins=["*"]` | Укажи конкретный домен своего клиента |
| **Rate limit по IP** | Ненадёжен если `X-Forwarded-For` подделан | Использовать за доверенным reverse proxy |

## 🔑 Управление ключами

Ключи хранятся в `data/keys.json` (автоматически при первом запуске). Файл имеет права `600`.

**Приоритет загрузки:**
```
.env → keys.json → авто-генерация
```

Ротация ADMIN_KEY (без сброса токенов):

```bash
curl -X POST "http://localhost:8000/admin/keys/rotate?rotate_admin=true" \
     -H "x-admin-key: ТЕКУЩИЙ_ADMIN_KEY"
```

Ротация SECRET_KEY (⚠️ все пользователи будут разлогинены):

```bash
curl -X POST "http://localhost:8000/admin/keys/rotate?rotate_secret=true" \
     -H "x-admin-key: ВАШ_ADMIN_KEY"
```
